# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in CIPHER Linux, please report it responsibly:

- **Do not** open a public issue for security vulnerabilities
- Instead, report it privately by opening a [GitHub Security Advisory](https://github.com/cipher-linux/cipher-linux/security/advisories/new) on this repository
- Include as much detail as possible: steps to reproduce, affected versions, potential impact

## What to Expect

- We'll acknowledge your report as soon as possible
- We'll investigate and keep you updated on progress
- Once fixed, we'll credit you in the release notes (unless you prefer to stay anonymous)

## Supported Versions

CIPHER Linux is currently in early development. Security policy and supported version table will be updated once the first stable release is published.

## Scope

This policy covers the CIPHER Linux distro build, packaging scripts, and official tooling in this organization. It does not cover upstream Debian/Kali packages — please report those to their respective maintainers.
